## How update_cpi.py works

### Pipeline (in order)

1. **`rebuild_source()`** ‚Äî runs `build_us_cpi.py` as subprocess ‚Üí `US_CPI.xlsx`.
2. **Load SOURCE** ‚Äî reads raw index and weights (`data_only=True`).
3. **Python variations** ‚Äî compute MoM, MM3MA, MM6MA, YoY in Python. Fallback for the new month not yet cached in Excel.
4. **`_excel_recalculate(FINAL_XLS)`** *(pre-read)* ‚Äî Excel COM forces recalculation and save. Restores the formula cache that openpyxl clears on every save.
5. **Read variation blocks** ‚Äî `data_only=True` reads cached values from formula block rows (392‚Äì774, 1163‚Äì1545, 1549‚Äì1931, 1935‚Äì2317).
6. **`_merge_variations()`** ‚Äî Excel values override Python column-by-column. New month (not yet cached) comes from Python.
7. **Load FINAL normally** ‚Äî for writing.
8. **`copy_new_months_to_final()`** ‚Äî appends new month column(s) to raw rows 6‚Äì391. Formula blocks extended via `Translator` (never overwritten with values).
9. **Write D3** ‚Äî `_label_to_date(latest_label)` with `number_format = "DD/MM/YYYY"`.
10. **`fill_tabela()`** ‚Äî writes cols C‚ÄìK for all 160 data rows.
11. **`apply_colors()`** ‚Äî colors col K: red = above avg, blue = below. Cap at 2œÉ per category.
12. **`wb_final.save()`** ‚Äî saves FINAL.xlsx (clears formula cache).
13. **`_excel_recalculate(FINAL_XLS)`** *(post-save)* ‚Äî restores formula cache again.
14. **`generate_html()`** ‚Äî writes `cpi_table.html`.
15. **`git_push()`** ‚Äî commits `cpi_table.html` + `CLAUDE.md` and pushes to `origin master`.

### Excel COM recalculation

`_excel_recalculate(xls_path)` uses `win32com.client`:
- Reuses an already-open Excel instance (safe alongside other open files).
- If Excel is not open, launches it hidden and quits when done.
- Called **before** reading formula blocks AND **after** every `wb.save()`.

### Category matching

- `match_name()`: case-insensitive prefix match, shortest candidate wins.
- `_build_cat_to_row()`: maps category ‚Üí **first** occurrence row (raw rows 6‚Äì391 only). Formula rows 392+ are ignored ‚Äî same category names reappear there.

## Color scale (column K)

Symmetric diverging gradient, capped per-category at 2œÉ:

- **Positive** (above avg) ‚Üí red: `FFFF{g:02X}{b:02X}`, `g = b = round(255 * (1 ‚àí intensity))`
- **Negative** (below avg) ‚Üí blue: `FF{r:02X}{r:02X}FF`, `r = round(255 * (1 ‚àí intensity))`
- **Zero** ‚Üí no fill
- `intensity = min(1.0, |dist| / (2 √ó œÉ of that category's full historical series))`

## Architecture of build_us_cpi.py

1. **`load_mapping`** ‚Äî reads `cpi_mapping.csv`: indent, category, series_id, basis.
2. **`fetch_all`** ‚Äî BLS API v2, batched (50 series / 20 years per request). 16 requests.
3. **`fetch_relative_importance`** ‚Äî BLS `aspects=True` for December RI. 16 requests.
4. **`build_workbook`** ‚Äî writes `US_CPI.xlsx` (sheets: `US_CPI`, `US_CPI_Weights`).

### cpi_mapping.csv format

| Column | Description |
|---|---|
| `indent` | 0‚Äì7, hierarchy depth |
| `category` | Display name as published by BLS |
| `series_id` | `CUSR0000*` = SA (black), `CUUR0000*` = NSA (blue) |
| `basis` | `SA` or `NSA` |

### Key constants in build_us_cpi.py

- `END_YEAR` ‚Äî update each January; the API caps at last published month.
- `API_KEY` ‚Äî BLS v2 registered key (500 req/day).
- `START_YEAR` ‚Äî currently 2000; col 4 = Jan-2000.

### BLS access constraints

`download.bls.gov` returns 403 for all scripts. Only `api.bls.gov/publicAPI/v2/timeseries/data/`
works programmatically. Relative Importance available from December 2012 via `aspects=True`.
The schedule page (`www.bls.gov/schedule/...`) also blocks automated access ‚Äî use a browser.

## Common failure modes

| Symptom | Cause | Fix |
|---|---|---|
| `PermissionError` on FINAL.xlsx | Excel has the file open | Close Excel, re-run |
| Formula blocks read as 0 categories | openpyxl cleared formula cache; COM unavailable | Install `pywin32`; ensure Excel is installed |
| `build_us_cpi.py` HTTP 429 | BLS API rate limit (500/day) | Wait 24h or use a different API key |
| All Excel variation blocks return `None` | COM recalculation failed silently | Run Excel manually, open FINAL.xlsx, save, then re-run script |
| New month not added to FINAL | `build_us_cpi.py` may not have fetched it yet | Check `_last_date_col` output; re-run after BLS publishes |
| `schtasks` date error on PT-BR Windows | Wrong date format | Script uses `DD/MM/YYYY` ‚Äî if failing, check locale with `Get-Culture` |
