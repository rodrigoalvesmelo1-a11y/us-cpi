## Project overview

Two-file workflow that produces a monthly US CPI analysis spreadsheet and HTML dashboard:

| File | Location | Purpose |
|---|---|---|
| `build_us_cpi.py` | repo root | Fetches raw CPI-U index and weights from BLS API; outputs `US_CPI.xlsx` |
| `update_cpi.py` | repo root | Monthly updater: rebuilds `US_CPI.xlsx`, fills the Tabela sheet in `US_CPI_FINAL.xlsx`, generates `cpi_table.html`, pushes to GitHub |
| `schedule_setup.py` | repo root | Creates Windows Task Scheduler tasks for each BLS release date |
| `US_CPI_FINAL.xlsx` | `C:\Users\rodri\CPI\` | Main deliverable: raw data + Tabela summary with colors |
| `cpi_table.html` | repo root (tracked) | HTML view of the Tabela, generated by `update_cpi.py` |

## Running the full monthly update

```
cd C:\Users\rodri\us-cpi
python update_cpi.py
```

This takes ~4 minutes (32 BLS API requests) and automatically:
1. Fetches latest raw data ‚Üí `US_CPI.xlsx`
2. Computes MoM / MM3MA / MM6MA / YoY variations in Python
3. Updates `C:\Users\rodri\CPI\US_CPI_FINAL.xlsx` (Tabela + colors)
4. Generates `cpi_table.html`
5. Commits and pushes to GitHub

## Running the data build only

## BLS CPI release schedule

CPI is published once a month at **8:30 AM Eastern Time**.
Release dates for 2025‚Äì2026: see `schedule_setup.py`.
Full schedule: https://www.bls.gov/schedule/news_release/cpi.htm (requires browser).

## Setting up the scheduler (first time)

```
cd C:\Users\rodri\us-cpi
python schedule_setup.py
```

Adjust `LOCAL_HOUR` / `LOCAL_MINUTE` in `schedule_setup.py` to match your timezone.
BLS releases at 8:30 AM ET. If you are in BRT (UTC-3, same as EDT+1h): BLS = 9:30 AM BRT.

## Color scale

Column K in the Tabela uses a symmetric diverging gradient:

- **Positive** distance (above historical average) ‚Üí red, `FFFF{g:02X}{b:02X}`
- **Negative** distance (below historical average) ‚Üí blue, `FF{r:02X}{r:02X}FF`
- **Zero** ‚Üí no fill
- **Scale cap** = 2 √ó œÉ of the section's distances. At |dist| ‚â• 2œÉ ‚Üí maximum color.

## Architecture of build_us_cpi.py

1. **`load_mapping`** ‚Äî reads `cpi_mapping.csv`: indent, category, series_id, basis.
2. **`fetch_all`** ‚Äî BLS API v2, batched (max 50 series, 20 years per request). 16 requests.
3. **`fetch_relative_importance`** ‚Äî BLS `aspects=True` for December RI. 16 requests.
4. **`build_workbook`** ‚Äî writes `US_CPI.xlsx` with two sheets.

## cpi_mapping.csv format

| Column | Description |
|---|---|
| `indent` | 0‚Äì7, hierarchy depth |
| `category` | Display name as published by BLS |
| `series_id` | `CUSR0000*` = SA (black), `CUUR0000*` = NSA (blue) |
| `basis` | `SA` or `NSA` |

## Key constants

In `build_us_cpi.py`:
- `END_YEAR` ‚Äî update each year; the API silently caps at the last published month
- `API_KEY` ‚Äî BLS v2 registered key (500 req/day)
- `START_YEAR` ‚Äî currently 2000; shifts all column positions

## BLS access constraints

`download.bls.gov` returns 403 for scripts. Only the API at
`api.bls.gov/publicAPI/v2/timeseries/data/` works programmatically.
Relative Importance data available from December 2012 onward via `aspects=True`.
