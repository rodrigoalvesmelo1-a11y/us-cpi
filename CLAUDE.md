## US_CPI_FINAL.xlsx structure

### Sheets

| Sheet | Description |
|---|---|
| `US_CPI` | Raw CPI-U index values (rows 6–391) + Excel formula blocks for variations (rows 392+) |
| `US_CPI_Weights` | Annual Relative Importance (BLS December RI, from December 2012) |
| `Tabela` | Summary: 4 sections × 40 categories = 160 data rows |

### US_CPI sheet layout

- **Row 5**: date headers (`Jan-00`, `Feb-00`, …). Col 4 = Jan-2000.
- **Rows 6–391**: raw CPI-U index values. Col C = category name, col D+ = monthly values.
- **Rows 392+**: Excel formula blocks — never overwrite with Python values.

| Metric | Row range |
|---|---|
| MoM (%) | 392–774 |
| MM3MA | 1163–1545 |
| MM6MA | 1549–1931 |
| YoY | 1935–2317 |

### Tabela sheet layout

| Section | Header row | Data rows | Metric |
|---|---|---|---|
| 1 | 3 | 4–43 | MoM |
| 2 | 45 | 46–85 | MM3MA |
| 3 | 87 | 88–127 | MM6MA |
| 4 | 129 | 130–169 | YoY |

Column layout per data row:

| Col | Content |
|---|---|
| B | Category name |
| C | Peso (Relative Importance, most recent month) — format `0.00` |
| D–I | Last 6 months of variation (newest = D) — format `0.00` |
| J | Média Histórica (all-time average of full series) — format `0.00` |
| K | Distância da Média (latest − average) — colored, format `0.00` |

**D3**: written as a real Excel date (`datetime.date`, day=1), number_format `DD/MM/YYYY`.
E3:I3 update automatically via Excel formulas.

## How update_cpi.py works

### Pipeline (in order)

1. **`rebuild_source()`** — runs `build_us_cpi.py` as subprocess → `US_CPI.xlsx`.
2. **Load SOURCE** — reads raw index and weights (`data_only=True`).
3. **Python variations** — compute MoM, MM3MA, MM6MA, YoY in Python. Fallback for the new month not yet cached in Excel.
4. **`_excel_recalculate(FINAL_XLS)`** *(pre-read)* — Excel COM forces recalculation and save. Restores the formula cache that openpyxl clears on every save.
5. **Read variation blocks** — `data_only=True` reads cached values from formula block rows (392–774, 1163–1545, 1549–1931, 1935–2317).
6. **`_merge_variations()`** — Excel values override Python column-by-column. New month (not yet cached) comes from Python.
7. **Load FINAL normally** — for writing.
8. **`copy_new_months_to_final()`** — appends new month column(s) to raw rows 6–391. Returns `(formula_lc, new_formula_cols)` for post-save COM extension.
9. **Write D3** — `_label_to_date(latest_label)` with `number_format = "DD/MM/YYYY"`.
10. **`fill_tabela()`** — writes cols C–K for all 160 data rows.
11. **`apply_colors()`** — colors col K: red = above avg, blue = below. Cap at 2σ per category.
12. **`wb_final.save()`** — saves FINAL.xlsx (clears formula cache).
13. **`_extend_and_recalculate(FINAL_XLS, formula_lc, new_formula_cols)`** *(post-save)* — uses Excel COM AutoFill to extend formula blocks to the new column, then recalculates and saves. Replaces the unreliable openpyxl Translator approach.
14. **`generate_html()`** — writes `cpi_table.html`.
15. **`git_push()`** — commits `cpi_table.html` + `CLAUDE.md` and pushes to `origin master`.

### Excel COM recalculation

`_excel_recalculate(xls_path)` uses `win32com.client`:
- Reuses an already-open Excel instance (safe alongside other open files).
- If Excel is not open, launches it hidden and quits when done.
- Called **before** reading formula blocks AND **after** every `wb.save()`.

### Category matching

- `match_name()`: case-insensitive prefix match, shortest candidate wins.
- `_build_cat_to_row()`: maps category → **first** occurrence row (raw rows 6–391 only). Formula rows 392+ are ignored — same category names reappear there.

## Color scale (column K)

Symmetric diverging gradient, capped per-category at 2σ:

- **Positive** (above avg) → red: `FFFF{g:02X}{b:02X}`, `g = b = round(255 * (1 − intensity))`
- **Negative** (below avg) → blue: `FF{r:02X}{r:02X}FF`, `r = round(255 * (1 − intensity))`
- **Zero** → no fill
- `intensity = min(1.0, |dist| / (2 × σ of that category's full historical series))`

## Architecture of build_us_cpi.py

1. **`load_mapping`** — reads `cpi_mapping.csv`: indent, category, series_id, basis.
2. **`fetch_all`** — BLS API v2, batched (50 series / 20 years per request). 16 requests.
3. **`fetch_relative_importance`** — BLS `aspects=True` for December RI. 16 requests.
4. **`build_workbook`** — writes `US_CPI.xlsx` (sheets: `US_CPI`, `US_CPI_Weights`).

### cpi_mapping.csv format

| Column | Description |
|---|---|
| `indent` | 0–7, hierarchy depth |
| `category` | Display name as published by BLS |
| `series_id` | `CUSR0000*` = SA (black), `CUUR0000*` = NSA (blue) |
| `basis` | `SA` or `NSA` |

### Key constants in build_us_cpi.py

- `END_YEAR` — update each January; the API caps at last published month.
- `API_KEY` — BLS v2 registered key (500 req/day).
- `START_YEAR` — currently 2000; col 4 = Jan-2000.

### BLS access constraints

`download.bls.gov` returns 403 for all scripts. Only `api.bls.gov/publicAPI/v2/timeseries/data/`
works programmatically. Relative Importance available from December 2012 via `aspects=True`.
The schedule page (`www.bls.gov/schedule/...`) also blocks automated access — use a browser.

## Common failure modes

| Symptom | Cause | Fix |
|---|---|---|
| `PermissionError` on FINAL.xlsx | Excel has the file open | Close Excel, re-run |
| Formula blocks read as 0 categories | openpyxl cleared formula cache; COM unavailable | Install `pywin32`; ensure Excel is installed |
| `build_us_cpi.py` HTTP 429 | BLS API rate limit (500/day) | Wait 24h or use a different API key |
| All Excel variation blocks return `None` | COM recalculation failed silently | Run Excel manually, open FINAL.xlsx, save, then re-run script |
| New month not added to FINAL | `build_us_cpi.py` may not have fetched it yet | Check `_last_date_col` output; re-run after BLS publishes |
| `schtasks` date error on PT-BR Windows | Wrong date format | Script uses `DD/MM/YYYY` — if failing, check locale with `Get-Culture` |
