## US_CPI_FINAL.xlsx structure

### Sheets

| Sheet | Description |
|---|---|
| `US_CPI` | Raw CPI-U index values (rows 6–391) + Excel formula blocks for variations (rows 392+) |
| `US_CPI_Weights` | Annual Relative Importance (BLS December RI, from December 2012) |
| `Tabela` | Summary: 4 sections × 40 categories = 160 data rows |

### US_CPI sheet layout

- **Row 5**: date headers (`Jan-00`, `Feb-00`, …). Col 4 = Jan-2000. Col index = `(year-2000)*12 + month + 3`.
- **Rows 6–391**: raw CPI-U index values. Col C = category name, col D+ = monthly values.
- **Rows 392+**: Excel formula blocks (never overwrite with Python values):

| Metric | Row range |
|---|---|
| Block 1 (intermediate) | 392–774 |
| MoM (%) | 392–774 |
| MM3MA | 1163–1545 |
| MM6MA | 1549–1931 |
| YoY | 1935–2317 |

Each formula block row: col C = category name, col D+ = formula values.

### Tabela sheet layout

| Section | Header row | Data rows | Metric |
|---|---|---|---|
| 1 | 3 | 4–43 | MoM |
| 2 | 45 | 46–85 | MM3MA |
| 3 | 87 | 88–127 | MM6MA |
| 4 | 129 | 130–169 | YoY |

Column layout per data row:

| Col | Content |
|---|---|
| B | Category name |
| C | Peso (Relative Importance, most recent month) — format `0.00` |
| D–I | Last 6 months of variation (newest = D) — format `0.00` |
| J | Média Histórica (all-time average of full series) — format `0.00` |
| K | Distância da Média (latest − average) — colored, format `0.00` |

**D3**: written by `update_cpi.py` as a real Excel date (`datetime.date`, day=1),
number_format `DD/MM/YYYY`. E3:I3 update automatically via Excel formulas.

## How update_cpi.py works

### Pipeline (in order)

1. **`rebuild_source()`** — runs `build_us_cpi.py` as subprocess to fetch latest BLS data → `US_CPI.xlsx`.

2. **Load SOURCE** — `load_workbook(SOURCE_XLS, data_only=True)`. Reads raw index and weights.

3. **Python variations** — compute MoM, MM3MA (3-month), MM6MA (6-month), YoY in Python from raw index. Used as fallback for the new month (not yet cached in Excel).

4. **`_excel_recalculate(FINAL_XLS)`** *(pre-read)* — opens FINAL.xlsx via Windows COM (`win32com.client`), forces `excel.Calculate()`, saves and closes. Restores the formula cache that openpyxl destroys on every save.

5. **Read variation blocks** — `load_workbook(FINAL_XLS, data_only=True)`. Reads cached formula values from rows 392–774 (MoM), 1163–1545 (MM3MA), 1549–1931 (MM6MA), 1935–2317 (YoY).

6. **`_merge_variations()`** — Excel-cached values override Python values column-by-column. The new month (not yet in Excel cache) comes from Python.

7. **Load FINAL normally** — `load_workbook(FINAL_XLS)` (not data_only) for writing.

8. **`copy_new_months_to_final()`** — appends new month column(s) to US_CPI and US_CPI_Weights raw data rows. Never touches rows 392+. Formula blocks extended via `Translator` (drag-to-fill equivalent).

9. **Write D3** — `ws_tabela.cell(3,4).value = _label_to_date(latest_label)` with `number_format = "DD/MM/YYYY"`.

10. **`fill_tabela()`** — writes cols C–K for all 160 data rows using merged variations.

11. **`apply_colors()`** — colors col K: red = above historical avg, blue = below. Intensity capped at 2σ of each category's full series.

12. **`wb_final.save()`** — saves FINAL.xlsx. This clears the Excel formula cache.

13. **`_excel_recalculate(FINAL_XLS)`** *(post-save)* — restores formula cache again.

14. **`generate_html()`** — writes `cpi_table.html` for GitHub Pages.

15. **`git_push()`** — `git add cpi_table.html CLAUDE.md` → commit → push to `origin master`.

### Excel COM recalculation

`_excel_recalculate(xls_path)` uses `win32com.client`:
- If Excel is already open: `GetActiveObject("Excel.Application")` (reuses it, does not quit).
- If not open: `Dispatch(...)` launches Excel hidden, quits when done.
- Call before reading formula blocks AND after every `wb.save()` — openpyxl clears cached
  formula values on every save, so without this step all formula cells read as `None`.

### Category matching

`match_name(target, source_cats)` does prefix-matching (case-insensitive, shortest match wins).
`_build_cat_to_row(ws)` maps category → **first** occurrence row number (raw data rows 6–391).
Formula block rows 392+ are skipped because the same category names reappear there.

## Color scale (column K)

Symmetric diverging gradient, per-category σ cap:

- **Positive** dist (above avg) → red: `FFFF{g:02X}{b:02X}` where `g = b = round(255*(1-intensity))`
- **Negative** dist (below avg) → blue: `FF{r:02X}{r:02X}FF` where `r = round(255*(1-intensity))`
- **Zero** → no fill
- `intensity = min(1.0, |dist| / (2 * σ_of_that_category's_full_series))`

## Architecture of build_us_cpi.py

1. **`load_mapping`** — reads `cpi_mapping.csv`: indent, category, series_id, basis.
2. **`fetch_all`** — BLS API v2, batched (max 50 series, 20 years per request). 16 requests.
3. **`fetch_relative_importance`** — BLS `aspects=True` for December RI. 16 requests.
4. **`build_workbook`** — writes `US_CPI.xlsx` with sheets `US_CPI` and `US_CPI_Weights`.

### cpi_mapping.csv format

| Column | Description |
|---|---|
| `indent` | 0–7, hierarchy depth |
| `category` | Display name as published by BLS |
| `series_id` | `CUSR0000*` = SA (black), `CUUR0000*` = NSA (blue) |
| `basis` | `SA` or `NSA` |

### Key constants in build_us_cpi.py

- `END_YEAR` — update each January; the API silently caps at the last published month.
- `API_KEY` — BLS v2 registered key (500 req/day).
- `START_YEAR` — currently 2000; shifts all column positions (col 4 = Jan-2000).

### BLS access constraints

`download.bls.gov` returns 403 for scripts. Only `api.bls.gov/publicAPI/v2/timeseries/data/`
works programmatically. Relative Importance available from December 2012 via `aspects=True`.

## Common failure modes

| Symptom | Cause | Fix |
|---|---|---|
| `PermissionError` on FINAL.xlsx | Excel has the file open | Close Excel, re-run |
| Formula blocks read as 0 categories | openpyxl cleared formula cache; COM not available | Install `pywin32`, ensure Excel is installed |
| `build_us_cpi.py` HTTP 429 | BLS API rate limit (500/day) | Wait 24h or use a different API key |
| `_merge_variations` shows 0 for all Excel blocks | COM recalculation failed | Check `win32com` install; run Excel manually and save |
| New month not added to FINAL | `_last_date_col` sees src_lc = final_lc | `build_us_cpi.py` may not have fetched the new month yet |
